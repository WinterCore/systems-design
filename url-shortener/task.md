# Exercise 1: URL Shortener (bit.ly)

## Question

Design a URL shortener (like bit.ly).

## Requirements

- Users can submit a long URL and get back a short URL
- Visiting the short URL redirects to the original
- Users can see how many times a short URL has been clicked
- Short URLs can optionally expire after a user-specified duration

## Deliverables

1. Database schema (tables, columns, types, indexes)
2. API endpoints (method, path, request/response bodies)
3. One paragraph explaining how you'd handle a short URL that gets 10,000 clicks per minute
